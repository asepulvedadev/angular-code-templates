🚀 Comenzar

Angular Code Templates es una herramienta CLI especializada diseñada para potenciar tu desarrollo Angular con Claude AI. Proporciona una colección curada de agentes, comandos, configuraciones, hooks e integraciones específicamente adaptadas para proyectos Angular.

💡

¿Qué es Angular Code Templates?

Angular Code Templates es un fork de Claude Code Templates, especializado para flujos de trabajo del ecosistema Angular. Proporciona agentes específicos de Angular, mejores prácticas de TypeScript, flujos de trabajo RxJS e integración con Angular CLI.

Requisitos Previos

  • Node.js 14+ instalado
  • Claude Code CLI instalado
  • Conocimiento básico de Angular y TypeScript
  • Un proyecto Angular (o listo para crear uno)

Inicio Rápido

bash 
# Instalar Angular Code Templates globalmente
npx angular-code-templates@latest

# O instalar componentes específicos
npx angular-code-templates@latest --agent angular/component-generator

📦 Instalación

Múltiples formas de instalar y usar Angular Code Templates en tus proyectos.

Instalación Interactiva

La forma más fácil de comenzar es usando el instalador interactivo:

bash 
npx angular-code-templates@latest

Instalar Componentes Específicos

bash 
# Instalar un agente
npx angular-code-templates@latest --agent angular/component-generator

# Instalar un comando
npx angular-code-templates@latest --command angular/generate-module

# Instalar un MCP
npx angular-code-templates@latest --mcp angular/nx-integration

# Instalar múltiples componentes a la vez
npx angular-code-templates@latest --agent angular/component-generator --command angular/generate-service --yes

Alcances de Instalación

Alcance Ubicación Caso de Uso
--user ~/.claude/ Disponible en todos tus proyectos
--project ./.claude/ Solo en el proyecto actual
Por defecto ./.claude/ Instalación a nivel de proyecto

🤖 Agentes

Los agentes son especialistas IA que te ayudan con tareas específicas de desarrollo Angular.

¿Qué son los Agentes?

Los agentes son asistentes IA especializados que entienden patrones de Angular, convenciones de TypeScript y flujos de trabajo RxJS. Cada agente es un experto en un dominio específico del desarrollo Angular.

Usando Agentes

bash 
# Instalar un agente
npx angular-code-templates@latest --agent angular/component-generator

# Usar el agente en Claude Code
# El agente se activará automáticamente cuando sea relevante

Agentes Angular Populares

🎨

Generador de Componentes

Se especializa en crear componentes Angular con mejores prácticas, incluyendo componentes standalone y signals.

--agent angular/component-generator
🔧

Especialista en Servicios

Experto en crear servicios Angular con inyección de dependencias, operadores RxJS y gestión de estado.

--agent angular/service-specialist
🗺️

Especialista en Rutas

Gestiona la configuración de rutas, lazy loading, guards y resolvers para aplicaciones Angular.

--agent angular/routing-specialist
⚠️

Mejores Prácticas de Agentes

  • Usa agentes específicos para tareas especializadas
  • Combina múltiples agentes para flujos de trabajo complejos
  • Revisa las sugerencias del agente antes de implementar
  • Personaliza los agentes según las necesidades de tu proyecto

⚡ Comandos

Comandos slash personalizados que puedes crear y usar en Claude Code para acciones rápidas.

Creando Comandos Personalizados

Los comandos son archivos markdown que definen comandos slash personalizados para Claude Code. Puedes crear tus propios comandos adaptados a tu flujo de trabajo Angular.

Estructura de Comandos

markdown 
# /generar-componente

Genera un nuevo componente Angular siguiendo las mejores prácticas.

## Propósito
Este comando crea un componente Angular con:
- Componente standalone (Angular 14+)
- Signals para reactividad
- Detección de cambios OnPush
- Modo estricto de TypeScript

## Uso
/generar-componente nombre-componente

## Implementación
1. Ejecutar ng generate component
2. Configurar como standalone
3. Añadir signals si es necesario
4. Configurar detección de cambios OnPush

## Ejemplos
- /generar-componente perfil-usuario
- /generar-componente shared/boton

Ubicación de Comandos

Crea tus comandos en:

bash 
.claude/commands/tu-comando.md
💡

Consejos para Comandos

  • Usa nombres descriptivos para tus comandos
  • Incluye ejemplos de uso en el archivo del comando
  • Documenta cualquier requisito previo o dependencia
  • Mantén los comandos enfocados en una sola tarea

⚙️ Configuraciones

Configura el comportamiento de Claude Code con configuraciones personalizadas para tus proyectos Angular.

Creando Configuraciones Personalizadas

Las configuraciones son archivos JSON que personalizan cómo funciona Claude Code en tu proyecto.

Estructura de Configuración

json 
{
  "name": "angular-typescript-strict",
  "description": "Configuración de modo estricto TypeScript para Angular",
  "category": "typescript",
  "settings": {
    "typescript.strict": true,
    "typescript.noImplicitAny": true,
    "typescript.strictNullChecks": true,
    "typescript.strictFunctionTypes": true,
    "angular.strictTemplates": true
  }
}

Statuslines con Scripts Python

Configuraciones especiales como statuslines pueden incluir scripts Python que se instalan automáticamente en .claude/scripts/

json 
{
  "name": "angular-build-monitor",
  "description": "Monitorear estado de build Angular en statusline",
  "category": "statusline",
  "script": "angular-build-monitor.py",
  "settings": {
    "statusline.enabled": true,
    "statusline.script": ".claude/scripts/angular-build-monitor.py"
  }
}

🪝 Hooks

Automatiza tareas con hooks basados en eventos que se activan en acciones específicas.

Creando Hooks Personalizados

Los hooks son activadores de automatización que ejecutan comandos shell en respuesta a eventos de desarrollo.

Estructura de Hooks

json 
{
  "name": "angular-lint-on-save",
  "description": "Ejecutar ESLint cuando se guardan archivos TypeScript",
  "type": "post-save",
  "event": "file-save",
  "pattern": "*.ts",
  "command": "ng lint --fix",
  "enabled": true,
  "async": true
}

Tipos de Hooks Disponibles

Tipo Se Activa Caso de Uso
pre-tool Antes de la ejecución de herramientas Validación, configuración
post-tool Después de la ejecución de herramientas Limpieza, registro
pre-commit Antes del commit de git Linting, pruebas
post-save Después de guardar archivo Auto-formato, compilación

🔌 MCPs (Model Context Protocol)

Integra servicios y herramientas externas con Claude Code a través de servidores MCP.

¿Qué son los MCPs?

Los MCPs (servidores Model Context Protocol) permiten a Claude Code interactuar con servicios externos como bases de datos, APIs y herramientas de desarrollo. Extienden las capacidades de Claude más allá de las operaciones de archivos.

Instalando MCPs

bash 
# Instalar un MCP
npx angular-code-templates@latest --mcp supabase

# Instalar múltiples MCPs
npx angular-code-templates@latest --mcp supabase --mcp github --yes

MCPs Populares para Angular

🗄️ Supabase

Operaciones de base de datos y autenticación

--mcp supabase

🐙 GitHub

Gestión de repositorios y CI/CD

--mcp github

📊 Firebase

Servicios backend y analytics

--mcp firebase

💻 Uso del CLI

Referencia completa de comandos y opciones del CLI de Angular Code Templates.

Sintaxis de Comandos

bash 
npx angular-code-templates@latest [opciones]

Opciones Disponibles

Opción Descripción Ejemplo
--agent <nombre> Instalar un agente --agent angular/component-generator
--command <nombre> Instalar un comando --command angular/generate-module
--setting <nombre> Instalar una configuración --setting typescript/strict-mode
--hook <nombre> Instalar un hook --hook git/pre-commit-lint
--mcp <nombre> Instalar un MCP --mcp supabase
--template <nombre> Instalar una plantilla --template typescript/angular-spa
--user Instalar en directorio de usuario --agent foo --user
--project Instalar en directorio del proyecto --agent foo --project
--yes Omitir confirmaciones --agent foo --yes
--dry-run Vista previa sin instalar --agent foo --dry-run

Ejemplos de Uso

bash 
# Modo interactivo (recomendado para principiantes)
npx angular-code-templates@latest

# Instalar un solo componente
npx angular-code-templates@latest --agent angular/component-generator

# Instalar múltiples componentes
npx angular-code-templates@latest --agent angular/component-generator --command angular/generate-service --yes

# Instalar en directorio de usuario (disponible en todos los proyectos)
npx angular-code-templates@latest --agent angular/component-generator --user

# Vista previa de instalación sin hacer cambios
npx angular-code-templates@latest --agent angular/component-generator --dry-run

# Instalar plantilla completa
npx angular-code-templates@latest --template typescript/angular-spa --yes

🔧 Solución de Problemas

Problemas comunes y sus soluciones.

❌ Los Componentes no Cargan

Problema: Los componentes instalados no aparecen en Claude Code.

Solución:

  1. Verifica que los archivos estén en la ubicación correcta (.claude/agents/, etc.)
  2. Reinicia Claude Code
  3. Verifica los permisos de archivo
  4. Revisa los registros de instalación en busca de errores

❌ El Comando NPX Falla

Problema: npx angular-code-templates@latest arroja un error.

Solución:

  1. Verifica la versión de Node.js (requiere 14+): node --version
  2. Limpia la caché de npm: npm cache clean --force
  3. Intenta con versión explícita: npx angular-code-templates@1.0.0

❌ Los Hooks no se Activan

Problema: Los hooks personalizados no se ejecutan.

Solución:

  1. Verifica enabled: true en la configuración del hook
  2. Comprueba que el patrón de eventos coincida con tus archivos
  3. Prueba el comando manualmente para asegurar que funciona
  4. Revisa la configuración de Claude Code para permisos de hooks

❌ Problemas de Conexión MCP

Problema: El servidor MCP no se puede conectar.

Solución:

  1. Verifica que las credenciales de API estén correctamente configuradas
  2. Comprueba la conectividad de red
  3. Revisa los registros del servidor MCP
  4. Asegúrate de que el servidor MCP esté en ejecución (para MCPs auto-hospedados)
